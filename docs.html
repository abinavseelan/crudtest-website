Where can crudTest help you?

Case study: Error-handling

Error handling is an integral part of any application. When making AJAX requests, it's usually hard and time-consuming to simulate error cases to fetch the error codes from your backend, and even harder to fetch the error codes from a third party service that you're using.

crudTest makes the process seamless. Just hit our APIs requesting the status code you need, and we'll respond with the requested status code so you can start building your error handling scenarios. üòÑ

Usage

Success Response Codes (2xx)

Endpoint: http://localhost:1337/status-code/2xx

(replace 'xx' with your desired response code)

Redirect Response Codes (3xx)

Endpoint: http://localhost:1337/status-code/3xx

(replace 'xx' with your desired response code)

Note: 3xx errors will trigger the catch block in a promise.

Client error Response Codes (4xx)

Endpoint: http://localhost:1337/status-code/4xx

(replace 'xx' with your desired response code)

Note: 4xx errors will trigger the catch block in a promise.

Server error Response Codes (5xx)

Endpoint: http://localhost:1337/status-code/5xx

(replace 'xx' with your desired response code)

Note: 5xx errors will trigger the catch block in a promise.
